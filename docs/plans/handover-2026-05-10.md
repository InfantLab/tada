## Repo housekeeping needed BEFORE writing v0.7.0 code

Three issues identified at end of last session, none addressed yet:

### 1. Broken symlink (blocks `git diff` cleanly)
```
specs/011-daily-timelines -> /workspaces/tada/specs/010-daily-timelines
```
A devcontainer/Codespaces absolute-path symlink that Windows can't follow. Causes `git diff` to abort partway with `Function not implemented`. The target `specs/010-daily-timelines` still exists, so the symlink is just a stale duplicate pointer.

**Fix:** `rm specs/011-daily-timelines` (it's a symlink so removing it is safe — doesn't touch the target).

### 2. EOL conversion churn (~50 spurious "modified" files)
`core.autocrlf=true` + no `.gitattributes` means Windows converts LF→CRLF on checkout. ~50 files show as modified that aren't really changed: `.github/agents/*`, `.github/prompts/*`, `.specify/scripts/*`, `.specify/templates/*`, `.specify/memory/constitution.md`, `app/pages/sessions.vue`, several scripts and spec plan files.

**Fix:** add a `.gitattributes` file that locks EOL handling per file type (recommended), then `git add --renormalize .` to bake the conversion in. Or change `core.autocrlf=input` (simpler, less robust). Either way, follow with `git checkout -- <file>` for any spuriously-modified files.

### 3. Untracked Claude state should be ignored
- `.claude/worktrees/` — per-machine worktree state, should be in `.gitignore`
- `app/public/freesounds.org/.claude/settings.local.json` and `app/public/sounds/.claude/settings.local.json` — local Claude settings, shouldn't be tracked

**Fix:** add to `.gitignore`:
```
.claude/worktrees/
**/.claude/settings.local.json
```

### 4. Possibly-real changes hiding behind the noise
Once #1 is fixed, run `git diff --stat -w` (ignore whitespace) to see what's *actually* different. Candidates that may have real changes:

- Binary files in `app/public/freesounds.org/`, `app/public/sounds/`, `app/public/screens/`, `design/screens/`

## Project context (for an agent who didn't see this conversation)

**Ta-Da!** is a Nuxt 3/4 + SQLite life-tracking PWA. Solo developer (Caspar). Hobby project, AGPL-3.0. Two related repos:

- [`InfantLab/tada`](https://github.com/InfantLab/tada) — main app, public, AGPL
- [`InfantLab/tada.living`](https://github.com/InfantLab/tada.living) — marketing/business planning, private

The app already ships a fairly mature feature set: six modules (Wins, Sessions, Moments, Tally, Rhythms, Timeline), voice AI extraction, weekly Monday celebration emails with four privacy tiers, 24-endpoint REST API, CSV/JSON/Markdown export, Obsidian sync, AGPL self-hostable Docker container.

The current pain that triggered this work: PWA meditation timer can't fire bells reliably when the phone is locked. That's a category-defining failure for a meditation app, so marketing was paused and the v0.7.0 plan was scoped to fix it via PWA hardening + native Android (Capacitor wrap, not rewrite).

The competitive read (in `tada.living/POSITIONING.md`) found a clear market gap between cold-punishment trackers (Streaks) and childlike companions (Finch) that no one credibly occupies — that's the position to claim.
